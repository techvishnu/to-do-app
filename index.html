Java Todo Application

Overview

The Java Todo Application is a simple command-line tool built using Core Java that allows users to manage their daily tasks effectively. The application provides functionalities to add, update, delete, and list tasks, offering a basic yet powerful interface for task management.

This project uses Java Serialization to save tasks to a file so that they can be persisted across sessions. It's structured around three main classes:

  • Task: Represents individual tasks.
  • TaskManager: Manages a list of tasks and handles saving/loading them to/from a file.
  • Main: The entry point of the application, providing a text-based user interface (CLI) to interact with the application.

Features

  • Add Tasks: Create a new task by providing a description and category (Personal, Work, Hobby, or Other).
  • Update Tasks: Modify an existing task's details, including its description, category, and completion status.
  • Delete Tasks: Remove a task from the list by its unique ID.
  • List Tasks: Display all tasks along with their ID, description, category, and completion status.
  • Persist Tasks: Save all tasks to a file (tasks.dat) and reload them when the application starts again.

  1. Running the Application
    • Open IntelliJ IDEA (or your preferred IDE) and load the project.
    • Make sure all required files (Task.java, TaskManager.java, Main.java, and Category.java) are correctly structured in the project.
    • Click the Run button in IntelliJ to compile and start the application.
    • You will be presented with a menu in the terminal/console:

File Structure

  • Task.java: Defines the task object with attributes like ID, description, category, and completion status.
  • TaskManager.java: Manages task operations such as adding, updating, deleting, and persisting tasks.
  • Main.java: The main class that provides a user interface to interact with the application.
  • Category.java: Enumeration defining the categories for tasks (PERSONAL, WORK, HOBBY, OTHER).

Data Persistence

Tasks are saved in a file named tasks.dat located in the project directory. The file ensures that tasks are persisted between sessions. When the application starts, it checks for this file and loads existing tasks.
